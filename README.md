# Задание 0.

### Текст задачи.

1. Опишите функцию nextch — модуль ввода-вывода. (Примечание: из-за отсутствия
анализатора, формирующего таблицу ошибок, создайте таблицу ошибок вручную.)
2. Разработайте набор тестов для тестирования модуля ввода-вывода.
3. Выполните тестирование модуля ввода-вывода.

### Алгоритм решения.

1. Создаю класс ReaderModule. В нем делаем поля _text (весь текст программы), _position (номер текущего символа), 
CurrentChar (ткеущий символ), Line (текущая строка) и Column (текущее расположение по горизонтали).
2. Теперь в том же классе добавляю конструктор SourceReader, который сохраняет в себе текст и таблицу ошибок
, а затем вызывает метод Nextch().
3. Далее, ключевой метод Nextch(), который читает каждый символ, меняя позицию, и проверяет на ошибки. 
4. Этим пунктом я создаю класс CompilerErrors, который хранит в себе расположения ошибки и ее название.
5. Затем создаю класс ErrorsTable, хранилище всех ошибок. В него добавляю поле _errors как список ошибок, 
метод Add() - для добавления ошибки, HasErrors() - для проверки на ошибку и Print() - для вывода всех ошибок. 
6. Здесь созаю новый класс Testing, в который вписываются тестове программы на pascal. Также после ввода сразу создается 
таблица ошибок и reader для чтения.
7. Последник класс Program, в котором я просто запускаю проект.
